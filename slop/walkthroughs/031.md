## Troubleshooting

### Issue 1: "Stats unavailable (no NVML)"

NVML can't initialize - driver or library issue.

**Fix:**

```bash
# Check if NVIDIA GPU is detected
lspci | grep NVIDIA

# Check if nvidia-smi works (verifies drivers)
nvidia-smi

# On Manjaro, install/reinstall drivers
sudo mhwd -a pci nonfree 0300

# Verify NVML library exists
ls /usr/lib/libnvidia-ml.so*
```

### Issue 2: GPU memory stays at 0%

ONNX Runtime isn't using CUDA provider.

**Fix:**

1. Check ONNX Runtime has CUDA support:
```bash
ls /opt/onnxruntime-gpu/lib/
# Should see: libonnxruntime_providers_cuda.so
```

2. Set library paths:
```bash
export LD_LIBRARY_PATH=/opt/onnxruntime-gpu/lib:/opt/cuda/lib64:$LD_LIBRARY_PATH
cargo test --test end_to_end_test -- --nocapture
```

3. Check `ort` crate features in `Cargo.toml`:
```toml
ort = { version = "2.0", features = ["cuda", "download-binaries"] }
```

### Issue 3: "CUDA error: out of memory"

Models are too large for GPU memory.

**Fix:**

Check GPU memory capacity:
```bash
nvidia-smi --query-gpu=memory.total --format=csv
```

Your RTX 3090 has 24GB, which is plenty. If you still get OOM:
- Process pages one at a time (already doing this)
- Check for memory leaks (use `nvml` to track memory growth)

### Issue 4: Compilation errors with nvml-wrapper

**Fix:**

Make sure you have the CUDA toolkit installed:
```bash
# Check CUDA version
nvcc --version

# On Manjaro
sudo pacman -S cuda
```
