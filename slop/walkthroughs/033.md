# Phase 1: Text Correction Foundation - Walkthrough

**Goal:** Fix OCR errors like `"who1have0beentogether20vearso"` → `"who have been together 20 years or"`
**Target:** 85-90% accuracy, <5ms overhead per page
**Time:** 3-4 hours total

---

## Mental Model: The Correction Pipeline

Think of OCR post-processing like editing a first draft:

1. **Normalization** - Fix encoding issues (é vs e + ´)
2. **Character fixes** - Fix obvious visual confusion (1→I in "1 am happy")
3. **Word splitting** - Fix missing spaces ("whohavebeen" → "who have been")
4. **Spelling** - Fix typos ("vearso" → "years or")

**Key insight:** These MUST happen in this order. You can't look up "1" in a dictionary - you must fix it to "I" first.

---

## Build Order

We'll implement three modules, each building on the last:

### Step 1: Character Confusion Module (45 min)
**What:** Fixes 1↔I, 0↔O, rn→m (visual similarity errors)
**Why first:** Dictionary lookups need real letters, not digits
**Pattern:** Context-aware character substitution

### Step 2: Symspell Integration (1.5-2 hours)
**What:** Word segmentation + spelling correction in one pass
**Why second:** Handles the majority of OCR errors (85-90%)
**Pattern:** Dictionary-based compound correction

### Step 3: Quality Measurement (45 min)
**What:** Calculate Character Error Rate (CER) on test corpus
**Why last:** Tells us if we need Stage 3 (analiticcl) or we're done
**Pattern:** Ground truth comparison

**Mental Model:** OCR engines confuse visually similar characters. "1" looks like "I", "0" looks like "O", "rn" looks like "m". We fix these BEFORE dictionary lookup.

**Critical pattern:** Context matters! "1" → "I" in "1 am happy", but "1" stays "1" in "I have 10 apples".

/// Calculate Character Error Rate (CER)
/// CER = (edit_distance / ground_truth_length) * 100
pub fn compute_cer(generated: &str, ground_truth: &str) -> f64 {
    let distance = levenshtein(generated, ground_truth);
    (distance as f64 / ground_truth.len().max(1) as f64) * 100.0
}

## Common Issues & Solutions

### Issue 1: "Dictionary not found"

**Problem:** Dictionaries not downloaded
**Solution:** Run `./scripts/download_dictionaries.sh`

### Issue 2: Tests fail with "no suggestion found"

**Problem:** symspell returns empty results
**Solution:** Check dictionary loaded correctly. Should see "Loaded 82765 words" in output.

### Issue 3: Too many false corrections

**Problem:** "The" → "They", "it" → "is"
**Solution:** Lower edit distance from 2 to 1 in `lookup_compound()`

### Issue 4: Slow performance

**Problem:** >5ms per text block
**Solution:** Verify dictionaries loaded ONCE at startup, not per-page
