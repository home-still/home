# Walkthrough 035: OCR Post-Correction Pipeline Integration

## Mental Model

OCR errors aren't random typos—they're systematic visual confusions. A human sees "rn" and "m" as different, but OCR sees similar pixel patterns. Your correction pipeline needs to mirror how an expert editor reviews OCR output:

1. **Character-level patterns first** - Fix systematic confusions (rn→m, cl→d)
2. **Word boundaries** - Separate "thequickbrown" or merge "comp lete"
3. **Dictionary validation** - Flag non-words for correction
4. **Context awareness** - Use surrounding words to pick "their" vs "there"
5. **Confidence scoring** - Know when to trust the correction

The critical insight: Process in stages from fast+certain to slow+contextual. Most text passes through quickly, difficult cases get expensive analysis.

## Current Pipeline Analysis

Your PDF-Mash pipeline (as of this walkthrough):

```
PDF → Images → Layout Detection → Reading Order (XY-Cut++)
    → Text Detection → Text Recognition (CTC) → [GAP] → Markdown
```

**The integration point**: Create a **separate PostProcessor** that runs after OCR but before markdown generation. This avoids breaking the OCREngine API and maintains separation of concerns.

**Why separate?** OCREngine should focus on raw text extraction. Correction logic belongs in its own module that can evolve independently.

use std::fs::File;
use std::io::{BufRead, BufReader};
use anyhow::{Result, Context};

const MAX_WORD_LENGTH: usize = 20;
const UNKNOWN_WORD_PENALTY: f64 = -20.0;

pub struct WordSegmenter {
    word_freq: HashMap<String, f64>,
    total_words: f64,
}

impl WordSegmenter {
    // TODO: Load word frequencies with proper error handling
    pub fn new(freq_file_path: &str) -> Result<Self> {
        let file = File::open(freq_file_path)
            .with_context(|| format!("Failed to open frequency file: {}", freq_file_path))?;

        let reader = BufReader::new(file);
        let mut word_freq = HashMap::new();
        let mut total = 0u64;

        // TODO: Parse frequency file
        // Format: "word count" (space or tab separated)
        // 1. Read each line
        // 2. Split by whitespace
        // 3. Parse word and count
        // 4. Accumulate total
        // 5. Store in HashMap

        for line in reader.lines() {
            let line = line?;
            let parts: Vec<&str> = line.split_whitespace().collect();

            if parts.len() >= 2 {
                let word = parts[0].to_lowercase();
                let count: u64 = parts[1].parse()
                    .with_context(|| format!("Invalid count in line: {}", line))?;

                word_freq.insert(word, count as f64);
                total += count;
            }
        }

        // Convert counts to log probabilities
        let total_f64 = total as f64;
        for (_word, freq) in word_freq.iter_mut() {
            *freq = (*freq / total_f64).ln();
        }

        Ok(Self {
            word_freq,
            total_words: total_f64,
        })
    }

    fn word_probability(&self, word: &str) -> f64 {
        self.word_freq
            .get(&word.to_lowercase())
            .copied()
            .unwrap_or(UNKNOWN_WORD_PENALTY)
    }

    // TODO: Implement DP segmentation with MAX_WORD_LENGTH optimization
    pub fn segment(&self, text: &str) -> Vec<String> {
        let n = text.len();
        if n == 0 {
            return vec![];
        }

        // DP arrays
        let mut opt = vec![f64::NEG_INFINITY; n + 1];
        let mut backtrack = vec![0; n + 1];
        opt[0] = 0.0;

        // TODO: Fill DP table with MAX_WORD_LENGTH constraint
        for j in 1..=n {
            let start = j.saturating_sub(MAX_WORD_LENGTH);

            for i in start..j {
                // IMPORTANT: Use char boundaries, not byte indices
                // TODO: Implement safe substring extraction

                // For now, simplified (you'll refine this):
                if let Some(word) = text.get(i..j) {
                    let score = opt[i] + self.word_probability(word);
                    if score > opt[j] {
                        opt[j] = score;
                        backtrack[j] = i;
                    }
                }
            }
        }

        // TODO: Backtrack to reconstruct words
        let mut words = Vec::new();
        let mut pos = n;

        while pos > 0 {
            let prev = backtrack[pos];
            if let Some(word) = text.get(prev..pos) {
                words.push(word.to_string());
            }
            pos = prev;
        }

        words.reverse();
        words
    }

    // TODO: Merge over-segmented fragments
    pub fn merge_fragments(&self, words: Vec<String>) -> Vec<String> {
        let mut result = Vec::new();
        let mut i = 0;

        while i < words.len() {
            if i + 1 < words.len() {
                // Try merging adjacent words
                let merged = format!("{}{}", words[i], words[i + 1]);
                let merged_prob = self.word_probability(&merged);
                let separate_prob = self.word_probability(&words[i])
                                  + self.word_probability(&words[i + 1]);

                if merged_prob > separate_prob {
                    result.push(merged);
                    i += 2;
                    continue;
                }
            }

            result.push(words[i].clone());
            i += 1;
        }

        result
    }
}

use anyhow::{Result, Context};

pub struct DictionaryCorrector {
    symspell: SymSpell<AsciiStringStrategy>,
}

impl DictionaryCorrector {
    // TODO: Initialize SymSpell with proper error handling
    pub fn new(dictionary_path: &str, max_edit_distance: i64) -> Result<Self> {
        let mut symspell: SymSpell<AsciiStringStrategy> = SymSpell::default();

        // Load dictionary
        symspell.load_dictionary(
            dictionary_path,
            0,  // term_index
            1,  // count_index
            " " // separator
        ).with_context(|| format!("Failed to load dictionary: {}", dictionary_path))?;

        Ok(Self { symspell })
    }

    // TODO: Correct single word with edit distance constraint
    pub fn correct_word(&self, word: &str, max_edit_distance: i64) -> Option<String> {
        if word.is_empty() {
            return None;
        }

        // Lookup with SymSpell
        let suggestions = self.symspell.lookup(
            word,
            Verbosity::Closest,  // Only return closest matches
            max_edit_distance,
        );

        // Return first (highest frequency) suggestion if different from input
        suggestions.first().and_then(|suggestion| {
            if suggestion.term != word {
                Some(suggestion.term.clone())
            } else {
                None
            }
        })
    }

    // TODO: Correct full text (word by word)
    pub fn correct(&self, text: &str) -> String {
        text.split_whitespace()
            .map(|word| {
                // Try edit distance 1 first (fast)
                self.correct_word(word, 1)
                    // If no match, try edit distance 2 (slower)
                    .or_else(|| self.correct_word(word, 2))
                    // Keep original if no correction found
                    .unwrap_or_else(|| word.to_string())
            })
            .collect::<Vec<_>>()
            .join(" ")
    }
}

## Known Dragons (Critical Pitfalls)

### 1. **Order Matters - Can't Skip Steps**
❌ **Don't** apply dictionary correction before segmentation
✅ **Do** fix character confusions → segment → dictionary → context

**Why:** Can't lookup "thequickbrown" in dictionary until segmented.

### 2. **Over-correction Destroys Valid Text**
❌ **Don't** aggressively correct titles, names, and technical terms
✅ **Do** use layout context to adjust correction threshold

**Why:** "Nguyen" is a valid surname, "PyTorch" is a valid library name.

### 3. **Performance Bottlenecks**
❌ **Don't** load dictionaries per-page or per-element
✅ **Do** initialize once, reuse across all text

**Why:** Dictionary loading takes seconds. Correction should be microseconds/word.

### 4. **Confidence Blindness**
❌ **Don't** blindly apply all corrections without tracking confidence
✅ **Do** track confidence and flag low-confidence changes for review

**Why:** Low confidence = higher error rate. Manual review catches mistakes.

### 5. **Unicode Handling Panics**
❌ **Don't** use byte indices for string slicing
✅ **Do** use char boundaries (`.chars()`, `.char_indices()`)

**Why:** Rust strings are UTF-8. Slicing mid-character = panic.

**Correct example:**
```rust
// ❌ WRONG - can panic on Unicode
let word = &text[i..j];  // If i or j is mid-character

// ✅ RIGHT - safe
let chars: Vec<char> = text.chars().collect();
let word: String = chars[i..j].iter().collect();

// ✅ BETTER - use char_indices()
let char_indices: Vec<(usize, char)> = text.char_indices().collect();
```

### 6. **Word Boundary Errors**
❌ **Don't** correct across word boundaries
✅ **Do** check if characters are in same word

**Example:**
```rust
// ❌ WRONG
"turn out" → "tum out"  // "rn" crossed boundary

// ✅ RIGHT
"learning" → "learning"  // "rn" within word → corrected to "m"
"turn out" → "turn out"  // "rn" crosses boundary → unchanged
```
