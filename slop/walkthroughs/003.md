
**What `--bin` means:** Creates a binary crate (produces an executable). Alternative is `--lib` (produces a library only).

We want **both** a library (for reusable components) and a binary (for CLI/server), so we'll create `src/lib.rs` manually later.

**Understanding Key Dependencies:**

1. **ort (ONNX Runtime):**
   - **Why "load-dynamic"?** Uses the system-installed ONNX Runtime from Walkthrough 01. Alternative is "download-binaries" which downloads ~100MB each time.
   - **CUDA feature:** Enables GPU acceleration. Without this, all inference runs on CPU (10x slower).

2. **pdfium-render:**
   - **Why not poppler-rs?** Pdfium is faster (2-3x) and handles more edge cases.
   - **thread_safe feature:** Critical for async server - multiple requests can process PDFs concurrently.

3. **axum vs actix-web:**
   - Axum: More ergonomic, better type safety, easier to learn
   - Actix: Slightly faster but more complex
   - **Choice:** Axum for maintainability

4. **anyhow vs thiserror:**
   - anyhow: Application-level errors (easy to use, context-rich)
   - thiserror: Library-level errors (strongly typed, for public APIs)
   - **We use both:** anyhow in `main.rs` and pipeline, thiserror in `lib.rs`

### 4.2: Create src/models/mod.rs

```bash
cat > src/models/mod.rs << 'EOF'
//! ML Model Inference Modules
//!
//! This module contains wrappers for all machine learning models:
//! - Layout detection: Identifies regions (text, table, formula, image)
//! - OCR: Extracts text from detected text regions
//! - Table extraction: Converts table regions to structured data
//! - Formula recognition: Converts math formulas to LaTeX
//!
//! Each model wrapper:
//! 1. Loads ONNX model via ort
//! 2. Handles preprocessing (image â†’ tensor)
//! 3. Runs GPU inference
//! 4. Handles postprocessing (tensor â†’ structured output)

pub mod layout;
pub mod ocr;
pub mod formula;
pub mod table;

use anyhow::Result;

/// Common trait for all ML models
///
/// This trait ensures consistent interface across all models.
/// Makes it easy to swap models or add new ones.
pub trait ModelInference {
    /// Input type (usually an image or crop)
    type Input;
    
    /// Output type (bounding boxes, text, etc.)
    type Output;
    
    /// Run inference on input
    fn infer(&self, input: Self::Input) -> Result<Self::Output>;
    
    /// Get model name (for logging)
    fn name(&self) -> &str;
    
    /// Warm up model (run dummy inference to initialize GPU)
    fn warmup(&self) -> Result<()> {
        // Default implementation: no-op
        // Override if model needs warmup
        Ok(())
    }
}

/// GPU device configuration
#[derive(Debug, Clone, Copy)]
pub struct GpuConfig {
    /// Device ID (0 for first GPU, 1 for second, etc.)
    pub device_id: i32,
    
    /// Maximum GPU memory to use (bytes)
    /// None = unlimited
    pub memory_limit: Option<usize>,
    
    /// Enable TensorRT optimizations (faster but longer first run)
    pub use_tensorrt: bool,
}

impl Default for GpuConfig {
    fn default() -> Self {
        Self {
            device_id: 0,
            memory_limit: Some(6 * 1024 * 1024 * 1024), // 6GB
            use_tensorrt: false, // Disabled by default (safer)
        }
    }
}
EOF
```

```bash
cat > src/main.rs << 'EOF'
//! PDF-Mash CLI and Server
//!
//! Usage:
//!   pdf-mash convert input.pdf -o output.md
//!   pdf-mash server --port 8080

use clap::{Parser, Subcommand};
use anyhow::Result;

#[derive(Parser)]
#[command(author, version, about, long_about = None)]
struct Cli {
    #[command(subcommand)]
    command: Commands,
}

#[derive(Subcommand)]
enum Commands {
    /// Convert PDF to Markdown
    Convert {
        /// Input PDF file
        input: String,
        
        /// Output Markdown file
        #[arg(short, long)]
        output: Option<String>,
        
        /// DPI for rendering (default: 150)
        #[arg(long, default_value_t = 150)]
        dpi: u16,
    },
    
    /// Start REST API server
    Server {
        /// Port to listen on
        #[arg(short, long, default_value_t = 8080)]
        port: u16,
        
        /// Host to bind to
        #[arg(long, default_value = "0.0.0.0")]
        host: String,
    },
    
    /// Download required models
    DownloadModels {
        /// Force re-download even if models exist
        #[arg(long)]
        force: bool,
    },
}
```

## Understanding the Build System

### Cargo Features

Our `Cargo.toml` uses **features** to enable optional functionality:

```toml
ort = { version = "2.0", features = ["cuda", "load-dynamic"] }
```

**What are features?**
- Optional dependencies or capabilities
- Enabled with `cargo build --features cuda`
- Reduces binary size if you don't need certain features

**Our features:**
- `cuda`: GPU acceleration (essential for performance)
- `load-dynamic`: Use system ONNX Runtime (vs bundled)

### Build Profiles

We defined two profiles:

1. **Development (`dev`):**
   - No optimization (`opt-level = 0`)
   - Fast compile times (~2 minutes)
   - Slow runtime (~10x slower than release)
   - **Use for:** Rapid iteration, debugging

2. **Release (`release`):**
   - Maximum optimization (`opt-level = 3`)
   - Link-time optimization (`lto = "fat"`)
   - Slow compile times (~10 minutes first build)
   - Fast runtime (optimal performance)
   - **Use for:** Production, benchmarking

**Build commands:**
```bash
# Development build (fast compile, slow run)
cargo build

# Release build (slow compile, fast run)
cargo build --release

# Run in development mode
cargo run -- convert input.pdf

# Run in release mode
cargo run --release -- convert input.pdf
```

## Common Issues and Solutions

### Issue 1: "can't find crate for `ort`"

**Cause:** Dependency hasn't been downloaded yet.

**Solution:**
```bash
cargo update
cargo check
```

### Issue 2: Long build times

**Cause:** Link-time optimization in release mode.

**Solution:** Use dev builds during development:
```bash
cargo build  # Fast
cargo build --release  # Only for final testing
```

### Issue 3: "failed to run custom build command for `ort`"

**Cause:** ONNX Runtime not found or environment variables not set.

**Solution:**
```bash
# Check environment variables
echo $ORT_DYLIB_PATH
echo $ORT_STRATEGY

# Verify ONNX Runtime installed
ls /usr/local/lib/libonnxruntime.so

# Re-source bashrc
source ~/.bashrc
```
