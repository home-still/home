This walkthrough downloads all required ML models (~20GB) and prepares them for use with ONNX Runtime. You'll learn about model formats, ONNX conversion, and model management.

## Understanding Model Formats

### PyTorch (.pt, .pth)
- **What:** Native PyTorch model format
- **Pros:** Full flexibility, easy to train/fine-tune
- **Cons:** Requires PyTorch runtime, Python dependency, slower inference
- **Use case:** Training, research

### ONNX (.onnx)
- **What:** Open Neural Network Exchange format
- **Pros:** Runtime-agnostic, optimized inference, works with Rust
- **Cons:** Some operations may not convert, fixed input sizes
- **Use case:** Production deployment (our choice)

### Conversion Quality
When converting PyTorch â†’ ONNX, accuracy typically drops 0-2% due to:
- Operator differences (some PyTorch ops â†’ approximations)
- Quantization (optional, for smaller models)
- Precision changes (FP32 â†’ FP16)

**Target:** Keep accuracy within 1% of original

#!/usr/bin/env python3
"""Convert PaddleOCR models to ONNX"""

import subprocess
from pathlib import Path

ocr_dir = Path("./models/ocr")

# Models to convert
models = [
    ("ch_PP-OCRv4_det_infer", "detection"),
    ("ch_PP-OCRv4_rec_infer", "recognition"),
    ("ch_ppocr_mobile_v2.0_cls_infer", "classification")
]

for model_name, output_name in models:
    model_dir = ocr_dir / model_name
    output_dir = ocr_dir / f"{output_name}_onnx"
    output_dir.mkdir(exist_ok=True)
    
    print(f"Converting {model_name}...")
    
    cmd = [
        "paddle2onnx",
        "--model_dir", str(model_dir),
        "--model_filename", "inference.pdmodel",
        "--params_filename", "inference.pdiparams",
        "--save_file", str(output_dir / "model.onnx"),
        "--opset_version", "11",
        "--enable_onnx_checker", "True"
    ]
    
    result = subprocess.run(cmd, capture_output=True, text=True)
    
    if result.returncode == 0:
        print(f"âœ“ {output_name} converted successfully")
    else:
        print(f"âœ— Failed to convert {output_name}")
        print(result.stderr)

print("\nAll OCR models converted!")
EOF

#!/usr/bin/env python3
"""Verify all ONNX models are valid"""

import onnx
from pathlib import Path

base_dir = Path("./models")

# Find all ONNX files
onnx_files = list(base_dir.rglob("*.onnx"))

print(f"Found {len(onnx_files)} ONNX models\n")

errors = []

for onnx_file in sorted(onnx_files):
    print(f"Checking {onnx_file.relative_to(base_dir)}...", end=" ")
    
    try:
        # Load model
        model = onnx.load(str(onnx_file))
        
        # Validate model
        onnx.checker.check_model(model)
        
        # Get model info
        graph = model.graph
        inputs = [(i.name, i.type) for i in graph.input]
        outputs = [(o.name, o.type) for o in graph.output]
        
        print(f"âœ“ Valid")
        print(f"   Inputs: {len(inputs)}, Outputs: {len(outputs)}")
        
    except Exception as e:
        print(f"âœ— FAILED")
        print(f"   Error: {str(e)}")
        errors.append((onnx_file, str(e)))

print()

if errors:
    print(f"âŒ {len(errors)} model(s) failed validation:")
    for file, error in errors:
        print(f"  - {file}: {error}")
else:
    print("âœ… All models validated successfully!")
EOF

## Common Issues

### Issue 1: "No module named 'onnxruntime'"

**Solution:**
```bash
pip install onnxruntime-gpu
```

### Issue 2: CUDA not available in ONNX Runtime

**Solution:**
```bash
# Make sure you downloaded GPU version
pip uninstall onnxruntime onnxruntime-gpu
pip install onnxruntime-gpu --extra-index-url https://pypi.org/simple

# Verify
python -c "import onnxruntime as ort; print(ort.get_available_providers())"
```

### Issue 3: Model conversion fails

**Solution:**
- Check PyTorch version: `pip install torch==2.1.0`
- Update conversion tools: `pip install -U optimum[exporters]`
- Try converting on CPU first (remove .cuda() calls)

### Issue 4: Models too large for disk

**Solution:**
- Delete source PyTorch models after conversion
- Use quantized models (see advanced section below)
- Store models on external drive

## Advanced: Model Quantization

Quantization reduces model size by 75% with minimal accuracy loss:

```bash
pip install onnxruntime-tools

cat > quantize_models.py << 'EOF'
from onnxruntime.quantization import quantize_dynamic, QuantType
from pathlib import Path

models = [
    "models/layout/doclayout_yolo_docstructbench_imgsz1024.onnx",
    "models/table/detection_onnx/model.onnx",
]

for model_path in models:
    model_path = Path(model_path)
    output_path = model_path.with_stem(model_path.stem + "_int8")
    
    print(f"Quantizing {model_path.name}...")
    quantize_dynamic(
        model_input=str(model_path),
        model_output=str(output_path),
        weight_type=QuantType.QInt8
    )
    
    original_size = model_path.stat().st_size / (1024**2)
    quantized_size = output_path.stat().st_size / (1024**2)
    
    print(f"  Original: {original_size:.1f} MB")
    print(f"  Quantized: {quantized_size:.1f} MB")
    print(f"  Reduction: {(1 - quantized_size/original_size)*100:.1f}%")
EOF

python quantize_models.py
```

**Trade-off:** 4x smaller but 1-2% accuracy loss
