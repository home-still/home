## Mental Model: The GPU Problem

ONNX Runtime with CUDA works like this:

```
Initialize Session
    ↓
Try to enable CUDAExecutionProvider
    ↓
    ├─ Success: GPU is used
    └─ Failure: Silent fallback to CPU
            ↓
        Your code runs fine but 10x slower
```

**The problem:** ONNX Runtime fails silently. If CUDA provider can't initialize, it just uses CPU.

**Common causes:**
1. ONNX Runtime not compiled with CUDA support
2. CUDA libraries not found at runtime
3. GPU memory full
4. Incorrect CUDA version mismatch

**The Problem:**
```rust

```

`session.metadata()` returns `ModelMetadata` which doesn't implement `Debug`, so you can't print it with `{:?}`.

**The Solution:**

You have 3 options:

**Option A - Remove the line** (fastest):
Delete line 37 entirely. You don't really need this debug output.

**Option B - Print something else** (recommended):
```rust
println!("✓ Layout model loaded");
```

**Option C - Get useful info** (if you want details):
```rust
let metadata = session.metadata()?;
println!("✓ Layout model: {} inputs, {} outputs",
    metadata.inputs.len(),
    metadata.outputs.len());
```

**Your Move:**

Open `pdf-mash/src/models/layout.rs` and either:

## Phase 4: Debug GPU Issues

If GPU isn't being used, work through these:

### Debug Path A: CUDA Libraries Not Found

**Symptoms:** "Failed to load CUDA provider" or silent CPU fallback

**Check:**
```bash
echo $LD_LIBRARY_PATH | grep cuda
ls /opt/cuda/lib64/libcudart.so*
```

**Fix:**
```bash
export LD_LIBRARY_PATH=/opt/cuda/lib64:$LD_LIBRARY_PATH
export CUDA_PATH=/opt/cuda
```

Add to `~/.bashrc` to make permanent.

### Debug Path B: ONNX Runtime Needs GPU Version

**Symptoms:** No CUDA provider available

**Fix:**
```bash
# Download correct GPU version
cd /tmp
wget https://github.com/microsoft/onnxruntime/releases/download/v1.22.0/onnxruntime-linux-x64-gpu-1.22.0.tgz
tar -xzf onnxruntime-linux-x64-gpu-1.22.0.tgz
sudo cp -r onnxruntime-linux-x64-gpu-1.22.0/lib/* /usr/local/lib/
sudo ldconfig
```

### Debug Path C: Feature Flags Wrong

**Check Cargo.toml:**
```toml
ort = { version = "2.0", features = ["cuda", "load-dynamic"] }
```

Must have `cuda` feature enabled.

### Debug Path D: GPU Memory Full

**Check:**
```bash
nvidia-smi
```

Look at "Memory-Usage". If it's 24000/24576 MB (full), other processes are hogging GPU.

**Fix:**
```bash
# Kill GPU processes
nvidia-smi --query-compute-apps=pid --format=csv,noheader | xargs kill
```

## Pattern You're Learning

**Silent fallbacks are dangerous** in ML systems:
- CUDA fails → CPU runs (10x slower)
- Model load fails → crashes (but late)
- Wrong input shape → garbage output (model runs fine)

**Always verify:**
1. Check providers after session creation
2. Monitor GPU during first runs
3. Validate outputs against ground truth

**This pattern applies to:**
- All ML inference systems
- Any performance-critical path with fallbacks
- Production deployments
