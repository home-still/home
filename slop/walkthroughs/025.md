# XY-Cut++ Paper Accuracy Fixes

## Mental Model

The current implementation works, but it's **over-engineered** compared to the paper. The authors of XY-Cut++ (Liu et al., 2025) ran grid search on 2.8k documents and found **exactly 4 weight combinations** that work best. We're using those 4 combinations plus 3 extra ones we invented. More code ≠ better results.

Think of it like a recipe: if the chef tested 4 ingredient ratios and published the best ones, don't add your own "improvements" without testing them on 2.8k dishes first.

**Paper Reference:** arXiv:2504.10258v1 - "XY-Cut++: Advanced Layout Ordering via Hierarchical Mask Mechanism"

---

We'll fix these in priority order:

1. **Equation 10 weights** - Core algorithm accuracy (highest impact)
