# OCR Post-Processing Integration Plan (Revised)

**Date:** November 19, 2025  
**Goal:** Fix word spacing and character errors in OCR output to produce readable markdown  
**Current Problem:** `"who1have0beentogether20vearso"` instead of `"who have been together 20 years or"`  
**Target:** 90-95% accuracy, <5ms overhead per page

---

## Mental Model: Why This Matters

You've built a complete PDF→Markdown pipeline with GPU acceleration. It works end-to-end:
- ✅ PDF parsing (pdfium)
- ✅ Layout detection (DocLayout-YOLO, GPU)
- ✅ Reading order (XY-Cut++)
- ✅ OCR text extraction (PaddleOCR, GPU)
- ✅ Markdown generation

**But the output is unreadable** because OCR engines fundamentally struggle with:

1. **Character confusion** - Visual similarity (I/1/l, O/0, rn/m, cl/d, vv/w)
2. **Word boundaries** - Cannot reliably detect spaces vs. wide character spacing
3. **Context ignorance** - Each character recognized independently

**This is NOT a model quality problem.** All OCR engines have this issue (Tesseract, PaddleOCR, EasyOCR). The solution is **post-processing**, not switching engines.

Think of it like speech recognition: The acoustic model outputs phonemes, but you need a language model to convert `"ice cream"` from `"I scream"`. Similarly, OCR outputs characters; post-processing converts them to meaningful text.

### Current Pipeline
```

                                                    ↑
                                            Returns: "who1have0been"
```

### Enhanced Pipeline (4-Stage Correction)
```
PDF → Pages → Layout Detection → Reading Order → OCR → POST-PROCESSING → Markdown
                                                            ↓

                                                    ↓
```

**Critical insight:** Pipeline order is **mandatory**, not just recommended. Analiticcl requires Unicode normalization first (documented requirement), and character confusion must be fixed before dictionary lookup.

## Strategic Decisions (Based on Research)

### Decision 1: Four-Stage Pipeline (Not Three)

**Research revealed:** Character confusion (1→I, 0→O) isn't handled by any dictionary-based tool. You need preprocessing.

**Pipeline stages:**
1. **Unicode Normalization (NFC)** - Ensures consistent character representation (mandatory for analiticcl)
2. **Character Confusion** - Context-aware substitution (1→I only when surrounded by letters)
3. **Spelling + Segmentation** - symspell's `lookup_compound()` handles both simultaneously
4. **OCR-Specific Patterns** - analiticcl for visual similarity errors (s→f, h→li, rn→m)

**Why this order:**
- Normalization must be first (analiticcl requirement)
- Character fixes before dictionary lookup (dictionaries don't contain "1" as a word)
- Segmentation + spelling together (symspell does both in one pass - 144x faster than separate tools)
- OCR patterns last (operates on partially-corrected text)

### Decision 2: Use Symspell's Integrated Approach (Remove wordninja-rs)

**Critical finding:** The original plan proposed wordninja-rs, but research revealed:
- wordninja-rs: 25.9ms per text block
- symspell.lookup_compound(): 0.18ms per text block
- **Speedup: 144x faster**

symspell's `lookup_compound()` does:
- Word segmentation (splits "whohavebeen" → "who have been")
- Spelling correction (fixes "vearso" → "years or")
- Compound awareness (handles multi-word corrections)
- All in a **single pass** with O(n) complexity

**Removing wordninja-rs eliminates:**
- Redundant processing (segment, then correct each word)
- Error propagation (segmentation errors affect correction)
- 25ms overhead per text block

### Decision 3: Start Minimal, Expand Based on Metrics

**Why:** Research shows algorithmic approaches achieve 85-92% accuracy at sub-millisecond speed. ML approaches add only 3-5% accuracy but cost 100-1000x more compute.

**Build order:**
1. **Phase 1 (MVP):** Character confusion + symspell → Fixes 85-90% of errors
2. **Phase 2 (Enhanced):** Add analiticcl for OCR patterns → Gets to 90-95%
3. **Phase 3 (Alternative):** Evaluate nlprule as potential replacement → 95%+ with grammar

**Each stage delivers independent value.**

### Decision 4: Two Paths Forward (Choose After Stage 1)

Research revealed **nlprule** as a comprehensive alternative to the multi-tool approach:

**Path A (Specialist):** Character confusion → symspell → analiticcl
- **Pros:** Fast (sub-millisecond), OCR-specific, deterministic, small memory (<50MB)
- **Cons:** No grammar correction, requires multiple tools

**Path B (Generalist):** Character confusion → nlprule
- **Pros:** Spelling + grammar + context, single tool, production-proven (cargo-spellcheck)
- **Cons:** Heavier (50-100MB), slower startup (5-10s), not OCR-specific

**Decision point:** Implement Path A (Stage 1-2), measure CER. If >10%, try Path B (Stage 4).

### Decision 5: Prioritize Performance AND Accuracy

**Why:** You're processing 3+ pages at ~1-2 seconds per page with GPU. Adding 100ms+ of post-processing would double processing time.

**Strategy:**
- Use algorithmic approaches (sub-millisecond core)
- Load dictionaries once at startup (amortize 1-5s cost)
- Process text blocks in parallel with Rayon (CPU while GPU does next page)
- Reserve GPU for actual OCR inference

**Target performance:**
- Character confusion: <1μs per character
- Symspell correction: <200μs per word  
- Full pipeline: <5ms per text block (50 words)
- **Total overhead per page: <5ms** (50 text blocks × 0.1ms)

```rust
use std::collections::HashMap;

/// Context-aware character confusion handler for OCR
/// Fixes: 1↔I↔l, 0↔O, rn→m, cl→d, vv→w
pub struct ConfusableMapper {
    // Bigram confusion patterns
    bigram_map: HashMap<String, String>,
}

impl ConfusableMapper {
    pub fn new() -> Self {
        let mut bigram_map = HashMap::new();
        
        // Common OCR bigram errors (visual similarity)
        bigram_map.insert("rn".to_string(), "m".to_string());
        bigram_map.insert("cl".to_string(), "d".to_string());
        bigram_map.insert("vv".to_string(), "w".to_string());
        bigram_map.insert("li".to_string(), "h".to_string());
        
        Self { bigram_map }
    }
    
    /// Fix character confusion with context awareness
    /// Only replaces digits when surrounded by letters
    pub fn deconfuse(&self, text: &str) -> String {
        // First pass: bigram substitution
        let mut result = text.to_string();
        for (pattern, replacement) in &self.bigram_map {
            result = result.replace(pattern, replacement);
        }
        
        // Second pass: context-aware single character
        let chars: Vec<char> = result.chars().collect();
        let mut fixed = String::with_capacity(result.len());
        
        for (i, &ch) in chars.iter().enumerate() {
            let prev = if i > 0 { Some(chars[i - 1]) } else { None };
            let next = if i < chars.len() - 1 { Some(chars[i + 1]) } else { None };
            
            let corrected = match ch {
                // 1 → I only when surrounded by letters
                '1' if prev.is_some_and(|c| c.is_alphabetic()) 
                    && next.is_some_and(|c| c.is_alphabetic()) => 'I',
                
                // 0 → O only when surrounded by letters  
                '0' if prev.is_some_and(|c| c.is_alphabetic()) 
                    && next.is_some_and(|c| c.is_alphabetic()) => 'O',
                
                // Keep original for numbers in numeric context
                _ => ch,
            };
            
            fixed.push(corrected);
        }
        
        fixed
    }
}
```

```rust
    confusables: ConfusableMapper,
    symspell: SymSpell,
    enabled: bool,
}

impl TextCorrector {
    /// Initialize with symspell dictionaries
    /// This takes 1-5 seconds - should be called ONCE at startup
    pub fn new(
        dict_path: &str,
        bigram_path: &str,
    ) -> Result<Self> {
        println!("Loading text correction dictionaries...");
        
        // Stage 0: Character confusion mapper (instant)
        let confusables = ConfusableMapper::new();
        
        // Stage 2: Symspell with bigram support
        let mut symspell = SymSpell::default();
        
        // Load unigram dictionary
        if std::path::Path::new(dict_path).exists() {
            symspell.load_dictionary(dict_path, 0, 1, " ")?;
            println!("  ✓ Loaded {} words", symspell.word_count());
        } else {
            anyhow::bail!("Dictionary not found: {}", dict_path);
        }
        
        // Load bigram dictionary for compound correction
        if std::path::Path::new(bigram_path).exists() {
            symspell.load_bigram_dictionary(bigram_path, 0, 2, " ")?;
            println!("  ✓ Loaded bigram dictionary");
        } else {
            eprintln!("⚠️  Bigram dictionary not found (compound correction disabled)");
        }
        
        Ok(Self {
            confusables,
            symspell,
            enabled: true,
        })
    }
    
    /// Main correction pipeline (4 stages)
    pub fn correct(&self, text: &str) -> Result<CorrectedText> {
        if !self.enabled || text.is_empty() {
            return Ok(CorrectedText {
                text: text.to_string(),
                confidence: 1.0,
                edits_made: 0,
                character_fixes: 0,
                word_fixes: 0,
            });
        }
        
        // Stage 1: Unicode normalization (NFC form)
        // CRITICAL: This MUST be first (analiticcl requirement)
        let normalized: String = text.nfc().collect();
        
        // Stage 2: Character confusion (1→I, 0→O, rn→m, etc.)
        let deconfused = self.confusables.deconfuse(&normalized);
        let character_fixes = if deconfused != normalized { 1 } else { 0 };
        
        // Stage 3: Spelling + segmentation (symspell compound)
        // This handles BOTH "whohavebeen" → "who have been"
        // AND "vearso" → "years or" in ONE PASS
        let suggestions = self.symspell.lookup_compound(
            &deconfused,
            2,  // max edit distance (2 is recommended for OCR)
        );
        
        let (corrected, confidence, word_fixes) = if let Some(suggestion) = suggestions.first() {
            let edits = suggestion.distance as usize;
            let confidence = if edits == 0 {
                1.0
            } else {
                // Simple confidence: fewer edits = higher confidence
                1.0 - (edits as f32 * 0.1).min(0.5)
            };
            
            (suggestion.term.clone(), confidence, edits)
        } else {
            // No correction found - return deconfused text
            (deconfused.clone(), 0.5, 0)
        };
        
        Ok(CorrectedText {
            text: corrected,
            confidence,
            edits_made: character_fixes + word_fixes,
            character_fixes,
            word_fixes,
        })
    }
    
    /// Enable/disable correction (for CLI flag --no-correction)
    pub fn set_enabled(&mut self, enabled: bool) {
        self.enabled = enabled;
    }
}
```

```rust
/// Calculate Character Error Rate (CER)
/// CER = edit_distance / ground_truth_length
pub fn compute_cer(generated: &str, ground_truth: &str) -> f64 {
    let distance = levenshtein(generated, ground_truth);
    (distance as f64 / ground_truth.len().max(1) as f64) * 100.0
}

/// Simple Levenshtein distance implementation
fn levenshtein(s1: &str, s2: &str) -> usize {
    let len1 = s1.chars().count();
    let len2 = s2.chars().count();
    let mut matrix = vec![vec![0; len2 + 1]; len1 + 1];

    for i in 0..=len1 {
        matrix[i][0] = i;
    }
    for j in 0..=len2 {
        matrix[0][j] = j;
    }

    for (i, c1) in s1.chars().enumerate() {
        for (j, c2) in s2.chars().enumerate() {
            let cost = if c1 == c2 { 0 } else { 1 };
            matrix[i + 1][j + 1] = (matrix[i][j + 1] + 1)
                .min(matrix[i + 1][j] + 1)
                .min(matrix[i][j] + cost);
        }
    }

    matrix[len1][len2]
}
```

## Error Handling Strategy

### Graceful Degradation (CRITICAL)

Post-processing must **NEVER** crash the pipeline. If correction fails, return original text:

```rust
impl TextCorrector {
    pub fn correct(&self, text: &str) -> Result<CorrectedText> {
        // Wrap entire correction pipeline in error handler
        match self.try_correct(text) {
            Ok(corrected) => {
                // Validate correction before returning
                if self.is_valid_correction(text, &corrected) {
                    Ok(corrected)
                } else {
                    eprintln!("⚠️  Invalid correction, returning original");
                    Ok(CorrectedText::unchanged(text))
                }
            }
            Err(e) => {
                eprintln!("⚠️  Correction failed: {}", e);
                Ok(CorrectedText::unchanged(text))
            }
        }
    }
    
    fn is_valid_correction(&self, original: &str, corrected: &CorrectedText) -> bool {
        // Length shouldn't change drastically
        let len_ratio = corrected.text.len() as f32 / original.len().max(1) as f32;
        if len_ratio < 0.3 || len_ratio > 3.0 {
            eprintln!("⚠️  Suspicious length change: {} → {}", 
                     original.len(), corrected.text.len());
            return false;
        }
        
        // Character set should remain similar (allow some expansion)
        let orig_chars: std::collections::HashSet<_> = original.chars().collect();
        let corr_chars: std::collections::HashSet<_> = corrected.text.chars().collect();
        let new_chars = corr_chars.difference(&orig_chars).count();
        if new_chars > original.len() / 2 {
            eprintln!("⚠️  Too many new characters added");
            return false;
        }
        
        // Word count shouldn't change by >50%
        let orig_words = original.split_whitespace().count();
        let corr_words = corrected.text.split_whitespace().count();
        let word_ratio = corr_words as f32 / orig_words.max(1) as f32;
        if word_ratio < 0.5 || word_ratio > 2.0 {
            eprintln!("⚠️  Suspicious word count change: {} → {}", 
                     orig_words, corr_words);
            return false;
        }
        
        true
    }
}

impl CorrectedText {
    /// Create unchanged result (for error cases)
    pub fn unchanged(text: &str) -> Self {
        Self {
            text: text.to_string(),
            confidence: 1.0,  // High confidence in original
            edits_made: 0,
            character_fixes: 0,
            word_fixes: 0,
        }
    }
}
```

```rust
impl Processor {
    pub fn process_pdf_streaming(&mut self, pdf_path: &str) -> Result<String> {
        let mut markdown = String::new();
        let total_pages = self.pdf_parser.page_count(pdf_path)?;
        
        for page_idx in 0..total_pages {
            // Process one page at a time
            let page = self.pdf_parser.parse_page(pdf_path, page_idx, 150)?;
            let page_md = self.process_page(&page)?;
            
            markdown.push_str(&page_md);
            markdown.push_str("\n\n");
            
            // Explicitly drop page data (releases memory)
            drop(page);
            
            // Yield to OS every 10 pages (prevents memory pressure)
            if page_idx % 10 == 0 {
                std::thread::yield_now();
            }
            
            // Optional: Log memory usage
            if page_idx % 50 == 0 {
                eprintln!("  Processed {}/{} pages", page_idx + 1, total_pages);
            }
        }
        
        Ok(markdown)
    }
}
```

## Risk Mitigation

### Risk 1: Symspell Dictionaries Too Large
**Concern:** 82,765 words + 243,342 bigrams might be too much memory

**Mitigation:**
- English dictionaries are only 5-50MB (verified)
- Load once at startup, share via Arc<> if needed
- For embedded systems, use smaller dictionary (40K words)

**Fallback:** If memory is truly constrained (<100MB available), use unigram-only (82K words, ~5MB)

### Risk 2: False Corrections on Correct Text
**Concern:** "The" → "They" because symspell is too aggressive

**Mitigation:**
- Symspell has configurable edit distance (use 2, not 3)
- Only corrects if dictionary match found with higher frequency
- Validation checks prevent drastic changes

**Monitoring:**
```rust
if corrected.edits_made > 0 && corrected.confidence < 0.7 {
    eprintln!("⚠️  LOW CONFIDENCE:");
    eprintln!("     Original:  {}", original);
    eprintln!("     Corrected: {}", corrected.text);
}
```

### Risk 3: Accuracy Still Poor After Stage 1
**Concern:** symspell alone doesn't reach 85-90% on your PDFs

**Mitigation:** That's why we have Stage 3 (analiticcl) and Stage 4 (nlprule)

**Decision tree:**
- CER < 10%: **DONE** (Stage 1 sufficient)
- CER 10-20%: Add Stage 3 (analiticcl for OCR patterns)
- CER > 20%: Evaluate Stage 4 (nlprule for comprehensive correction)

### Risk 4: Performance Impact
**Concern:** Correction adds significant latency

**Mitigation:**
- Symspell is genuinely sub-millisecond (0.03-0.18ms)
- Character confusion is microseconds
- Even with 100 text blocks per page: 100 × 0.2ms = 20ms
- Your OCR takes 8-1000ms per region - 20ms is 2-20% overhead

**Validation:**
```rust

let elapsed = start.elapsed();

    eprintln!("⚠️  Slow correction: {:?} for {} chars", elapsed, text.len());
}
```

```rust
impl Processor {
    fn collect_error_patterns(&self) -> Vec<(String, String)> {
        // Collect (wrong, correct) pairs from corrections
        self.correction_history
            .iter()
            .filter(|c| c.confidence > 0.9)  // High confidence only
            .map(|c| (c.original.clone(), c.corrected.clone()))
            .collect()
    }
    
    fn retrain_analiticcl(&self) -> Result<()> {
        let patterns = self.collect_error_patterns();
        
        // Write to training file
        let mut file = File::create("data/learned_errors.tsv")?;
        for (wrong, correct) in patterns {
            writeln!(file, "{}\t{}\t1", wrong, correct)?;
        }
        
        // Retrain model
        std::process::Command::new("analiticcl")
            .args(&["learn", "--input", "data/learned_errors.tsv",
                   "--output", "models/paddle_ocr_variants.bin"])
            .status()?;
        
        Ok(())
    }
}
```

## Key Reminders

### 1. Pipeline Order is Mandatory

```
✅ CORRECT:
Unicode Norm → Char Confusion → OCR Patterns → Spell/Segment

❌ WRONG:
Spell/Segment → Char Confusion
(Dictionary doesn't have "1" as a word!)

❌ WRONG:
OCR Patterns → Unicode Norm
(Analiticcl requires NFC form!)
```

### 2. Load Dictionaries Once

```
✅ CORRECT:
Processor::new() {
    let corrector = TextCorrector::new(...)?;  // Once at startup
}

❌ WRONG:
process_page() {
    let corrector = TextCorrector::new(...)?;  // Per-page = disaster!
}
```

### 3. Graceful Degradation Always

```
✅ CORRECT:
match corrector.correct(text) {
    Ok(c) => c,
    Err(_) => CorrectedText::unchanged(text),  // Return original
}

❌ WRONG:
let corrected = corrector.correct(text)?;  // Pipeline crashes if correction fails
```

### 4. Symspell Does Segmentation

```
✅ CORRECT:
symspell.lookup_compound("whohavebeen")
// Returns: "who have been" (segmented + corrected)

❌ WRONG (old plan):
let segmented = wordninja.split("whohavebeen");  // 25.9ms
let corrected = symspell.lookup(segmented)?;     // + 0.18ms
// Total: 26ms (144x slower!)
```

### 5. Start Minimal, Expand Based on Metrics

```
✅ CORRECT:
Stage 1 → Measure CER → Decide if need Stage 3

❌ WRONG:
Implement all stages upfront (wasted effort if Stage 1 sufficient)
```
