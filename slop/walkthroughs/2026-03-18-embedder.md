# Walkthrough 4: embedder — GPU-Accelerated ONNX Embedding

## Goal

Embed chunks into 768-dimensional vectors using nomic-embed-text v1.5 via ONNX
Runtime with CUDA acceleration. This is the GPU-bound stage of the pipeline.

**Acceptance criteria:** `cargo check -p embedder` passes. (No runtime test
without the ONNX model file.)

---

## Workspace changes

Add to `[workspace.dependencies]` in root `Cargo.toml`:

```toml
ort = { version = "2.0.0-rc.12", features = ["cuda"] }
ndarray = "0.17"
```

(`tokenizers` should already be there from Walkthrough 3.)

### TODO: Implement these methods

1. **`new(model_path, tokenizer_id, batch_size) -> Result<Self>`**
   - Build an ORT `Session` with CUDA + CPU fallback:
     ```rust
     Session::builder()?
         .with_execution_providers([
             CUDAExecutionProvider::default().build(),
             CPUExecutionProvider::default().build(),
         ])?
         .commit_from_file(model_path)?;
     ```
   - Load tokenizer via `Tokenizer::from_pretrained`
   - Set `max_tokens = 512`

2. **`embed_chunks(&mut self, chunks: Vec<Chunk>) -> Result<Vec<EmbeddedChunk>>`**
   - Sort chunks by text length (reduces padding waste)
   - Prepend `"search_document: "` to each text
   - Process in batches of `self.batch_size`
   - Restore original order after embedding
   - Return `EmbeddedChunk` with chunk + embedding vector

3. **`embed_texts(&mut self, texts: &[String]) -> Result<Vec<Vec<f32>>>`**
   - Tokenize batch with `self.tokenizer.encode_batch`
   - Build padded `input_ids` and `attention_mask` as `Vec<i64>`
   - Create ORT tensors via `Tensor::from_array`
   - Run session with `ort::inputs!` macro
   - Handle two output shapes:
     - 3D `[batch, seq, hidden]` → mean pooling with attention mask
     - 2D `[batch, hidden]` → already pooled
   - L2-normalize all vectors

### Key patterns

**Creating ORT tensors:**
```rust
let input_ids_tensor = Tensor::from_array((
    vec![batch_size as i64, max_len as i64],
    input_ids.into_boxed_slice(),
))?;
```

**Running the model:**
```rust
let outputs = self.session.run(ort::inputs![
    "input_ids" => input_ids_tensor,
    "attention_mask" => attention_mask_tensor,
])?;
```

**Extracting output:**
```rust
let (shape, raw_data) = outputs[0].try_extract_tensor::<f32>()?;
let dims: &[i64] = &**shape;
```

**Mean pooling (3D output):**
For each item in batch, sum hidden dims over only the real tokens (not padding),
divide by token count, then L2-normalize.

**L2 normalization:**
```rust
let norm: f32 = embedding.iter().map(|x| x * x).sum::<f32>().sqrt();
if norm > 0.0 {
    for v in &mut embedding { *v /= norm; }
}
```

### Dragon

`ort` v2.0.0-rc.12 is a **release candidate**. The version must be specified
exactly — `ort = "2"` will not resolve to it.

### Dragon

`Session::run()` requires `&mut self` in ort v2. Your `embed_chunks` and
`embed_texts` methods must take `&mut self`, not `&self`.

### Dragon

The model output may be 3D (needs pooling) or 2D (already pooled) depending on
the ONNX export. Handle both cases or you'll get shape mismatches at runtime.

### Dragon

nomic-embed-text requires `"search_document: "` prefix on all input texts for
document embedding. Query-time uses `"search_query: "` instead. This is not
optional — embeddings without the prefix will have poor retrieval quality.
