# Walkthrough 3: chunker — Token-Aware Text Splitting

## Goal

Split `AcademicDocument`s into embedding-ready `Chunk`s using a real HuggingFace
tokenizer. Chunks are 512 tokens max with 50-token overlap and sentence boundary
detection.

**Acceptance criteria:** `cargo test -p chunker` — 3 tests pass.

---

Add to `[workspace.dependencies]` in root `Cargo.toml`:

## Step 2: `src/lib.rs` — The Chunker

Constants:

```rust
const MAX_TOKENS: usize = 512;   // nomic context is 8192, but 512 is optimal for retrieval
const OVERLAP_TOKENS: usize = 50;
```

### Struct

```rust
pub struct Chunker {
    tokenizer: Tokenizer,
}
```

### TODO: Implement these methods

1. **`new(tokenizer_path: &str) -> Result<Self>`** — load tokenizer from file
2. **`from_pretrained(model_id: &str) -> Result<Self>`** — load from HuggingFace hub
3. **`chunk_document(&self, doc: &AcademicDocument) -> Result<Vec<Chunk>>`**
   - For abstract: build text as `"{title}\n\n{abstract}"`, split into chunks
   - Title-only docs: single chunk with just the title
   - No title and no abstract: return empty vec
   - Each chunk carries all metadata fields from the doc
4. **`chunk_body(&self, doc, section, body_text) -> Result<Vec<Chunk>>`**
   - CCH format: `"{title} > {section}\n\n{text}"`
   - Account for header tokens in the budget (`MAX_TOKENS - header_tokens`)
   - Skip if available budget < 50 tokens
5. **`split_text(&self, text, max_tokens, overlap) -> Result<Vec<String>>`**
   - Tokenize with `self.tokenizer.encode(text, false)`
   - If total tokens <= max_tokens, return single chunk
   - Otherwise, iterate: find end of chunk, look back for sentence boundary,
     extract text span using token offsets, advance with overlap
6. **`find_sentence_boundary(&self, text, offsets, start, end) -> usize`**
   - Look back 20% of chunk size for `.`, `!`, or `?`
   - If no boundary found, return `end`
7. **`count_tokens(&self, text) -> Result<usize>`** — helper for header budget

### Key patterns

**Token offsets for text extraction:**
```rust
let encoding = self.tokenizer.encode(text, false)?;
let offsets = encoding.get_offsets(); // Vec<(usize, usize)> — (char_start, char_end)
let text_start = offsets[start].0;
let text_end = offsets[chunk_end - 1].1;
let chunk_text = &text[text_start..text_end];
```

**Infinite-loop prevention in overlap chunking:**
```rust
let advance = chunk_end.saturating_sub(overlap);
if advance <= start {
    start = chunk_end; // force progress
} else {
    start = advance;
}
```

**CCH (Contextual Chunk Headers):**
Prepending title + section to each chunk improves retrieval because the embedding
captures document-level context, not just the chunk text.

### Dragon

`tokenizers` crate needs the `http` feature for `from_pretrained`. Without it,
only `from_file` works and you'll get a compile error on the HTTP download path.

### Dragon

Tests should gracefully skip if the tokenizer download fails (CI environments
without internet). Pattern:
```rust
let chunker = Chunker::from_pretrained("nomic-ai/nomic-embed-text-v1.5");
if chunker.is_err() {
    eprintln!("Skipping: tokenizer not available");
    return;
}
```
