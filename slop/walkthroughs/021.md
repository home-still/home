## The Mental Model

### Why Visual Comparison Matters

Reading order is **spatial**. Terminal output shows Y coordinates:
```
1. title at y=187
2. text at y=301
3. text at y=334
```

But you can't see if that makes sense without looking at the page! Visual comparison lets you:
- **Verify sorting**: Do numbers flow top→bottom, left→right?
- **Debug errors**: Is box #5 in the wrong column?
- **Understand layout**: See multi-column detection in action

## Step 3: Implement the Drawing Logic

Now fill in the TODOs from Step 1b.

**My implementation:**

```rust
pub fn draw_bboxes_with_numbers(
    &self,
    image: &DynamicImage,
    bboxes: &[BBox],
) -> Result<RgbImage> {
    let mut annotated = image.to_rgb8();

    // Load embedded font
    let font_data = include_bytes!("../../assets/DejaVuSans.ttf");
    let font = Font::try_from_bytes(font_data as &[u8])
        .ok_or_else(|| anyhow::anyhow!("Failed to load embedded font"))?;

    for (idx, bbox) in bboxes.iter().enumerate() {
        let color = self.class_color(&bbox.class_name);

        // Draw rectangle (same as draw_bboxes)
        let rect = Rect::at(bbox.x1 as i32, bbox.y1 as i32)
            .of_size((bbox.x2 - bbox.x1) as u32, (bbox.y2 - bbox.y1) as u32);

        for offset in 0..3 {
            let expanded = Rect::at(rect.left() - offset, rect.top() - offset).of_size(
                rect.width() + 2 * offset as u32,
                rect.height() + 2 * offset as u32,
            );
            draw_hollow_rect_mut(&mut annotated, expanded, color);
        }

        // Draw white circle for number background
        let circle_x = (bbox.x1 + 25.0) as i32;
        let circle_y = (bbox.y1 + 25.0) as i32;
        draw_filled_circle_mut(&mut annotated, (circle_x, circle_y), 20, Rgb([255, 255, 255]));

        // Draw sequence number in black
        let number = format!("{}", idx + 1);
        let scale = Scale::uniform(32.0);

        draw_text_mut(
            &mut annotated,
            Rgb([0, 0, 0]),
            (bbox.x1 + 10.0) as i32,
            (bbox.y1 + 5.0) as i32,
            scale,
            &font,
            &number,
        );
    }

    Ok(annotated)
}
```

**Pattern Spotted**: The `include_bytes!` macro embeds the font file into the compiled binary at compile-time. No runtime file I/O needed!

## Common Mistakes

### Mistake 1: Font Path Wrong
```rust
// WRONG - absolute path
let font_data = include_bytes!("/home/user/pdf_masher/assets/DejaVuSans.ttf");

// RIGHT - relative to source file
let font_data = include_bytes!("../../assets/DejaVuSans.ttf");
```

The path is relative to `src/utils/visualization.rs`, so `../../` goes up to project root.

### Mistake 2: Saving After Sort Only
```rust
// WRONG - can't compare!
sort_reading_order(&mut bboxes);
if save_viz {
    visualizer.draw_bboxes(&page.image, &bboxes)?;
}

// RIGHT - save before AND after
if save_viz { /* before */ }
sort_reading_order(&mut bboxes);
if save_viz { /* after */ }
```

### Mistake 3: Number Positioning Off-Screen
```rust
// WRONG - number might be outside small boxes
let circle_x = (bbox.x1 + 100.0) as i32;  // Too far right!

// RIGHT - small offset from corner
let circle_x = (bbox.x1 + 25.0) as i32;
```
