# Walkthrough 01: System Setup on Manjaro Linux

## Overview

This walkthrough sets up your Manjaro Linux system with all dependencies needed for GPU-accelerated PDF processing. You'll install CUDA for GPU acceleration, ONNX Runtime for ML inference, and supporting libraries.

**Time Required:** 30-45 minutes  
**Skill Level:** Beginner-friendly  
**Prerequisites:** Manjaro Linux with NVIDIA GPU

---

## Understanding the Components

Before we start, let's understand what we're installing and **why**:

### 1. CUDA Toolkit
**What it is:** NVIDIA's parallel computing platform that lets programs use your GPU for computation.

**Why we need it:** All our ML models (layout detection, OCR, table extraction) run on the GPU for 10-20x speed improvements. Without CUDA, everything runs on CPU and is too slow for production use.

**Size:** ~3GB

### 2. cuDNN (CUDA Deep Neural Networks)
**What it is:** GPU-accelerated library of primitives for deep neural networks (convolutions, pooling, normalization).

**Why we need it:** ONNX Runtime uses cuDNN to accelerate neural network operations. It's the difference between 500ms and 50ms per page.

**Size:** ~500MB

### 3. ONNX Runtime
**What it is:** Cross-platform inference engine for running ONNX (Open Neural Network Exchange) models.

**Why we need it:** Our approach is to export PyTorch models to ONNX format and run them via ONNX Runtime in Rust. This gives us:
- Near-native C++ performance
- GPU acceleration
- No Python dependency
- Standardized model format

**Size:** ~100MB

### 4. Pdfium
**What it is:** Google Chrome's PDF rendering engine, extracted as a standalone library.

**Why we need it:** We need to:
- Render PDF pages to images (for ML processing)
- Extract text (for comparison/validation)
- Read PDF metadata (page count, author, etc.)

Pdfium is **much faster and more reliable** than alternatives like Poppler or MuPDF.

**Size:** ~50MB

## Step 1: Verify NVIDIA GPU

First, confirm you have an NVIDIA GPU and drivers installed.

```bash
# Check if nvidia-smi is available
nvidia-smi
```

**Expected output:**
```
+-----------------------------------------------------------------------------+
| NVIDIA-SMI 545.29.06    Driver Version: 545.29.06    CUDA Version: 12.3   |
|-------------------------------+----------------------+----------------------+
| GPU  Name        Persistence-M| Bus-Id        Disp.A | Volatile Uncorr. ECC |
| Fan  Temp  Perf  Pwr:Usage/Cap|         Memory-Usage | GPU-Util  Compute M. |
|===============================+======================+======================|
|   0  NVIDIA GeForce ...  Off  | 00000000:01:00.0  On |                  N/A |
|  0%   45C    P8    15W / 350W |    500MiB /  24576MiB|      2%      Default |
+-------------------------------+----------------------+----------------------+
```

**What to look for:**
- GPU name appears (any NVIDIA GPU with 6GB+ VRAM works)
- Driver version is shown (should be 525+)
- CUDA Version is listed (12.x preferred)

**If nvidia-smi fails:**
```bash
# Install NVIDIA drivers
sudo pacman -S nvidia nvidia-utils
sudo reboot
```

**Understanding VRAM:** The "Memory-Usage" line shows VRAM. You need **at least 6GB total VRAM** for all models to fit. More is better:
- 6GB: Basic operation (models run sequentially)
- 8GB: Comfortable (some parallel processing)
- 12GB+: Ideal (full parallel processing)

## Step 2: Update System

Always start with a fully updated system to avoid dependency conflicts.

```bash
# Update package database and all installed packages
sudo pacman -Syu
```

**What this does:**
- `-S`: Synchronize (install/update)
- `-y`: Refresh package database
- `-u`: Upgrade all packages

**If prompted to replace packages:** Say yes. This is normal when package maintainers reorganize dependencies.

**Time:** 5-10 minutes depending on how outdated your system is.

## Step 3: Install CUDA Toolkit

Manjaro has CUDA in its official repositories, making installation straightforward.

```bash
# Install CUDA toolkit (includes compiler, libraries, samples)
sudo pacman -S cuda

# Install CUDA command-line tools
sudo pacman -S cuda-tools
```

**What gets installed:**
- **nvcc:** NVIDIA CUDA Compiler (compiles CUDA code)
- **cuBLAS:** GPU-accelerated linear algebra
- **cuFFT:** GPU-accelerated Fast Fourier Transform
- **cuRAND:** GPU random number generation
- **CUDA libraries:** Runtime libraries our applications will use

**Installation location:** `/opt/cuda`

**Verify installation:**
```bash
# Check compiler version
nvcc --version
```

**Expected output:**
```
nvcc: NVIDIA (R) Cuda compiler driver
Copyright (c) 2005-2023 NVIDIA Corporation
Built on Wed_Sep_20_15:32:06_PDT_2023
Cuda compilation tools, release 12.3, V12.3.103
```

**Why version matters:** CUDA 12.x is preferred (newer, faster), but 11.8+ works fine. The key is that **ONNX Runtime must be compiled for your CUDA version** (we'll download the right one).

## Step 4: Install cuDNN

cuDNN provides optimized implementations of neural network operations.

```bash
# Install cuDNN (CUDA Deep Neural Network library)
sudo pacman -S cudnn
```

**What cuDNN provides:**
- Optimized convolution algorithms (5-10x faster than naive implementations)
- Pooling operations (max, average)
- Activation functions (ReLU, sigmoid, tanh)
- Batch normalization
- RNN/LSTM operations

**Why it's separate from CUDA:** cuDNN is proprietary (requires NVIDIA license agreement during install), while CUDA toolkit is more open.

**Verify installation:**
```bash
# Check if cuDNN library exists
ls -l /opt/cuda/lib64/libcudnn*
```

**Expected output:**
```
lrwxrwxrwx 1 root root libcudnn.so -> libcudnn.so.9
-rwxr-xr-x 1 root root libcudnn.so.9.0.0
```

## Step 5: Set Environment Variables

Programs need to know where CUDA libraries are installed.

```bash
# Open your bash configuration
nano ~/.bashrc
```

**Add these lines at the end:**
```bash
# CUDA Environment for PDF-Mash
export CUDA_HOME=/opt/cuda
export PATH=$CUDA_HOME/bin:$PATH
export LD_LIBRARY_PATH=$CUDA_HOME/lib64:$LD_LIBRARY_PATH
```

**What each variable does:**

1. **CUDA_HOME:** Base directory where CUDA is installed. Many build tools look for this.

2. **PATH:** Where the shell looks for executables. We add `$CUDA_HOME/bin` so we can run `nvcc` from anywhere.

3. **LD_LIBRARY_PATH:** Where programs look for shared libraries (.so files) at runtime. Without this, programs can't find CUDA libraries even if installed.

**Save and exit:** Press `Ctrl+X`, then `Y`, then `Enter`

**Apply changes immediately:**
```bash
source ~/.bashrc
```

**Test that it worked:**
```bash
# This should now work from any directory
nvcc --version

# Check that libraries can be found
echo $LD_LIBRARY_PATH
```

## Step 6: Install Build Tools

We need compilers and tools to build Rust programs with C/C++ dependencies.

```bash
# Install essential build tools
sudo pacman -S base-devel cmake git pkg-config

# Install OpenSSL (needed for HTTPS in REST API)
sudo pacman -S openssl

# Install Git LFS (for downloading large model files)
sudo pacman -S git-lfs
git lfs install
```

**What each package does:**

- **base-devel:** Meta-package containing gcc, g++, make, and other build essentials
- **cmake:** Cross-platform build system (many C++ libraries use it)
- **git:** Version control
- **pkg-config:** Helps compile software by locating installed libraries
- **openssl:** Cryptography library (HTTPS, TLS)
- **git-lfs:** Git extension for large files (ML models are 100MB-2GB each)

**Why Git LFS matters:** Regular git is slow with large binary files. Models from HuggingFace use git-lfs, so we need it to download them efficiently.

## Step 7: Install ONNX Runtime with GPU Support

This is the inference engine that will run our ML models.

```bash
# Change to temporary directory
cd /tmp

# Download ONNX Runtime with GPU support
wget https://github.com/microsoft/onnxruntime/releases/download/v1.22.0/onnxruntime-linux-x64-gpu-1.22.0.tgz

# Extract archive
tar -xzf onnxruntime-linux-x64-gpu-1.22.0.tgz

# Copy libraries to system location
sudo cp -r onnxruntime-linux-x64-gpu-1.22.0/lib/* /usr/local/lib/

# Copy header files (needed for Rust bindings to compile)
sudo cp -r onnxruntime-linux-x64-gpu-1.22.0/include/* /usr/local/include/

# Update shared library cache
sudo ldconfig
```

**Understanding the steps:**

1. **wget:** Downloads the pre-built binary package (saves 30+ minutes of compilation)

2. **tar -xzf:** Extracts the compressed archive
   - `-x`: Extract
   - `-z`: Decompress with gzip
   - `-f`: From file

3. **cp -r:** Copy recursively (includes subdirectories)

4. **ldconfig:** Rebuilds the cache of shared libraries. Without this, programs won't find `libonnxruntime.so` even though it's installed.

**Why /usr/local/lib?** This is the standard location for locally-installed libraries (not managed by package manager). Rust's build system automatically looks here.

**Verify installation:**
```bash
# Check library exists
ls -l /usr/local/lib/libonnxruntime.so*

# Check version
strings /usr/local/lib/libonnxruntime.so | grep "^1\."
```

**Expected output:**
```
-rw-r--r-- 1 root root 89M libonnxruntime.so.1.22.0
lrwxrwxrwx 1 root root libonnxruntime.so -> libonnxruntime.so.1.22.0
```

**Add environment variable for ONNX Runtime:**
```bash
echo 'export ORT_DYLIB_PATH=/usr/local/lib' >> ~/.bashrc
echo 'export ORT_STRATEGY=system' >> ~/.bashrc
source ~/.bashrc
```

**What these mean:**
- **ORT_DYLIB_PATH:** Tells Rust's `ort` crate where to find ONNX Runtime
- **ORT_STRATEGY=system:** Use system-installed ONNX Runtime (don't download it separately)

## Step 8: Install Pdfium

Pdfium handles all PDF operations (rendering, text extraction, metadata).

```bash
# Change to temporary directory
cd /tmp

# Download pre-built Pdfium for Linux
wget https://github.com/bblanchon/pdfium-binaries/releases/download/chromium%2F7350/pdfium-linux-x64.tgz

# Extract
tar -xzf pdfium-linux-x64.tgz

# Copy library to system location
sudo cp lib/libpdfium.so /usr/local/lib/

# Update library cache
sudo ldconfig
```

**Why this specific version?** Build 7350 corresponds to Chrome 131, which is stable and well-tested. Newer builds might have bugs; older builds might lack features.

**Verify installation:**
```bash
# Check library exists
ls -l /usr/local/lib/libpdfium.so

# Check symbols (functions) in library
nm -D /usr/local/lib/libpdfium.so | grep FPDF_InitLibrary
```

**Expected output:**
```
-rw-r--r-- 1 root root 45M /usr/local/lib/libpdfium.so
00000000001a2b40 T FPDF_InitLibrary
```

The `T` means the symbol is defined (good). If you see `U`, the library is broken.

## Step 10: Final System Verification

Let's verify everything is working together.

**Create test program:**
```bash
mkdir -p /tmp/test-cuda
cd /tmp/test-cuda
```

**Create test.cu:**
```bash
cat > test.cu << 'EOF'
#include <stdio.h>

__global__ void hello() {
    printf("Hello from GPU thread %d\n", threadIdx.x);
}

int main() {
    hello<<<1, 10>>>();
    cudaDeviceSynchronize();
    return 0;
}
EOF
```

**Compile and run:**
```bash
nvcc test.cu -o test
./test
```

**Expected output:**
```
Hello from GPU thread 0
Hello from GPU thread 1
Hello from GPU thread 2
...
Hello from GPU thread 9
```

**If this works:** Your CUDA installation is fully functional! 

**If it fails:** Check that:
1. nvidia-smi shows your GPU
2. CUDA_HOME is set correctly
3. You rebooted after installing drivers

## Common Issues and Solutions

### Issue 1: "CUDA driver version is insufficient"

**Cause:** GPU driver is too old for installed CUDA version.

**Solution:**
```bash
# Update drivers
sudo pacman -S nvidia
sudo reboot
```

### Issue 2: "libcudnn.so.9: cannot open shared object file"

**Cause:** cuDNN not installed or LD_LIBRARY_PATH not set.

**Solution:**
```bash
# Reinstall cuDNN
sudo pacman -S cudnn

# Check environment variable
echo $LD_LIBRARY_PATH | grep cuda
```

### Issue 3: "nvcc: command not found"

**Cause:** PATH not set correctly or CUDA not installed.

**Solution:**
```bash
# Verify CUDA is installed
ls /opt/cuda

# Re-add to PATH
echo 'export PATH=/opt/cuda/bin:$PATH' >> ~/.bashrc
source ~/.bashrc
```

### Issue 4: Downloads are slow

**Cause:** GitHub rate limiting or slow connection.

**Solution:**
```bash
# Use mirror for ONNX Runtime (if available)
# Or download via browser and copy to /tmp

# For Pdfium, consider alternate mirror:
wget https://github.com/bblanchon/pdfium-binaries/releases/download/chromium%2F7350/pdfium-linux-x64.tgz \
  --tries=5 --wait=3
```
