## Understanding PDF Structure

Before coding, let's understand what we're parsing:

### PDF Components

```

â”œâ”€â”€ Pages

â”‚   â”‚   â”œâ”€â”€ Content stream (text, vectors, images)

â”‚   â”‚   â”œâ”€â”€ Resources (fonts, images)

â”‚   â”œâ”€â”€ Page 2

â””â”€â”€ Attachments (embedded files)
```

**Key Concepts:**

1. **Points vs Pixels:**
   - PDFs use "points" (1 point = 1/72 inch)
   - Screens use pixels
   - **DPI (dots per inch)** converts: pixels = points Ã— DPI / 72
   - Example: A4 page at 72 DPI = 595Ã—842 pixels
   - Example: Same page at 150 DPI = 1240Ã—1754 pixels

2. **Why Render to Images?**
   - ML models need pixel data (images), not vector data (PDF)
   - Ensures consistent input format
   - Handles complex layouts (mixed fonts, graphics, etc.)

3. **Text Extraction:**
   - PDF stores text with positioning
   - Can extract without rendering (faster, but layout info limited)
   - Use for validation against OCR

## Step 1: Create PDF Parser Module

Open `src/pipeline/pdf_parser.rs`:

```rust
//! PDF Parsing Module
//!
//! Handles all PDF operations:
//! - Document loading and validation
//! - Page rendering to images
//! - Text extraction
//! - Metadata extraction
//!
//! Uses pdfium-render (Google Chrome's PDF engine)

use anyhow::{Context, Result};
use image::DynamicImage;
use pdfium_render::prelude::*;
use std::path::Path;
use serde::{Serialize, Deserialize};

/// PDF Parser
///
/// Thread-safe wrapper around Pdfium library.
/// Can be cloned and shared across async tasks.
pub struct PdfParser {
    pdfium: Pdfium,
}

impl PdfParser {
    /// Create new PDF parser
    ///
    /// Initializes the Pdfium library. This is relatively expensive,
    /// so create one instance and reuse it.
    pub fn new() -> Result<Self> {
        // Initialize Pdfium (loads library, sets up context)
        let pdfium = Pdfium::default();
        
        Ok(Self { pdfium })
    }

    /// Load PDF document from file path
    ///
    /// This validates the PDF and prepares it for rendering.
    /// The returned PdfDocument holds a lock on the file.
    pub fn load_document(&self, path: impl AsRef<Path>) -> Result<PdfDocument> {
        let document = self.pdfium
            .load_pdf_from_file(path, None)  // None = no password
            .context("Failed to load PDF file")?;
        
        Ok(document)
    }

    /// Load PDF from bytes (useful for API uploads)
    pub fn load_document_from_bytes(&self, data: &[u8]) -> Result<PdfDocument> {
        let document = self.pdfium
            .load_pdf_from_byte_slice(data, None)
            .context("Failed to load PDF from bytes")?;
        
        Ok(document)
    }
}

/// Metadata extracted from PDF
#[derive(Debug, Clone, Serialize, Deserialize)]
pub struct DocumentMetadata {
    /// Number of pages
    pub page_count: u16,
    
    /// Document title (if set)
    pub title: Option<String>,
    
    /// Document author (if set)
    pub author: Option<String>,
    
    /// Document subject (if set)
    pub subject: Option<String>,
    
    /// Creation date (if set)
    pub creation_date: Option<String>,
    
    /// Modification date (if set)
    pub modification_date: Option<String>,
    
    /// PDF version (e.g., "1.7")
    pub pdf_version: String,
    
    /// Is encrypted (password-protected)?
    pub is_encrypted: bool,
    
    /// Page dimensions (width, height in points)
    /// Stored per-page because pages can have different sizes
    pub page_sizes: Vec<(f32, f32)>,
}

impl PdfParser {
    /// Extract metadata from document
    pub fn extract_metadata(&self, document: &PdfDocument) -> Result<DocumentMetadata> {
        let pages = document.pages();
        let page_count = pages.len();
        
        // Extract page sizes
        let mut page_sizes = Vec::with_capacity(page_count as usize);
        for page in pages.iter() {
            let width = page.width().value;
            let height = page.height().value;
            page_sizes.push((width, height));
        }
        
        // Extract document info
        let metadata = DocumentMetadata {
            page_count,
            title: document.metadata().title().ok().map(|s| s.to_string()),
            author: document.metadata().author().ok().map(|s| s.to_string()),
            subject: document.metadata().subject().ok().map(|s| s.to_string()),
            creation_date: document.metadata().creation_date().ok().map(|d| d.to_string()),
            modification_date: document.metadata().modification_date().ok().map(|d| d.to_string()),
            pdf_version: document.version().to_string(),
            is_encrypted: false, // If we got here, it's decrypted or not encrypted
            page_sizes,
        };
        
        Ok(metadata)
    }
}

/// Single page data
#[derive(Debug, Clone)]
pub struct PageData {
    /// Page index (0-based)
    pub page_idx: usize,
    
    /// Rendered page image
    pub image: DynamicImage,
    
    /// Page width in points
    pub width_pt: f32,
    
    /// Page height in points
    pub height_pt: f32,
    
    /// Extracted text (optional, for validation)
    pub text: Option<String>,
}

impl PdfParser {
    /// Render single page to image
    ///
    /// # Arguments
    /// * `page` - PDF page to render
    /// * `dpi` - Resolution (dots per inch)
    ///
    /// Higher DPI = better quality but slower and more memory.
    /// Recommended: 150 DPI (balance of quality and speed)
    /// Max recommended: 300 DPI (for very small text)
    pub fn render_page(&self, page: &PdfPage, dpi: u16) -> Result<DynamicImage> {
        // Calculate target dimensions
        // Formula: pixels = points Ã— DPI / 72
        let width_px = (page.width().value * dpi as f32 / 72.0) as i32;
        let height_px = (page.height().value * dpi as f32 / 72.0) as i32;
        
        // Configure rendering
        let render_config = PdfRenderConfig::new()
            .set_target_width(width_px)
            .set_target_height(height_px)
            // Don't rotate landscape pages (keep original orientation)
            .rotate_if_landscape(PdfPageRenderRotation::None, false);
        
        // Render to bitmap
        let bitmap = page
            .render_with_config(&render_config)
            .context("Failed to render page")?;
        
        // Convert Pdfium bitmap to image::DynamicImage
        let image = bitmap.as_image();
        
        Ok(image)
    }

    /// Extract text from page
    ///
    /// This extracts text with positioning information.
    /// Useful for:
    /// - Validation against OCR
    /// - Copy-paste capability
    /// - Search functionality
    pub fn extract_page_text(&self, page: &PdfPage) -> Result<String> {
        let text_page = page.text().context("Failed to get text page")?;
        
        // Get all text (ignores formatting, just concatenates)
        let text = text_page.all();
        
        Ok(text)
    }

    /// Process entire PDF into pages
    ///
    /// This is the main entry point - converts PDF to structured data.
    ///
    /// # Arguments
    /// * `path` - Path to PDF file
    /// * `dpi` - Rendering resolution
    ///
    /// # Returns
    /// Vector of PageData structs, one per page
    pub fn parse_to_pages(
        &self,
        path: impl AsRef<Path>,
        dpi: u16,
    ) -> Result<Vec<PageData>> {
        // Load document
        let document = self.load_document(&path)?;
        
        // Get pages
        let pages = document.pages();
        let page_count = pages.len();
        
        tracing::info!("Processing {} pages at {} DPI", page_count, dpi);
        
        let mut page_data = Vec::with_capacity(page_count as usize);
        
        // Process each page
        for (idx, page) in pages.iter().enumerate() {
            tracing::debug!("Processing page {}/{}", idx + 1, page_count);
            
            // Render to image
            let image = self.render_page(&page, dpi)?;
            
            // Extract text (optional, for validation)
            let text = self.extract_page_text(&page).ok();
            
            // Create PageData
            let data = PageData {
                page_idx: idx,
                image,
                width_pt: page.width().value,
                height_pt: page.height().value,
                text,
            };
            
            page_data.push(data);
        }
        
        tracing::info!("Successfully processed {} pages", page_count);
        
        Ok(page_data)
    }

    /// Parse PDF with metadata
    ///
    /// Convenience method that returns both pages and metadata
    pub fn parse_with_metadata(
        &self,
        path: impl AsRef<Path>,
        dpi: u16,
    ) -> Result<(Vec<PageData>, DocumentMetadata)> {
        let document = self.load_document(&path)?;
        
        // Extract metadata
        let metadata = self.extract_metadata(&document)?;
        
        // Drop document reference so we can load it again for rendering
        // (pdfium-render locks the file during rendering)
        drop(document);
        
        // Parse pages
        let pages = self.parse_to_pages(&path, dpi)?;
        
        Ok((pages, metadata))
    }
}

// Implement Clone for PdfParser (thread-safe)
impl Clone for PdfParser {
    fn clone(&self) -> Self {
        // Create new Pdfium instance
        // Each clone gets its own library context
        Self::new().expect("Failed to clone PdfParser")
    }
}
```

**Key Design Decisions:**

1. **Why separate render and extract_text?**
   - Text extraction is fast (no rendering)
   - Use text for validation, image for ML
   - Can skip text extraction if not needed

2. **Why store points and pixels?**
   - Points: Original PDF coordinate system
   - Pixels: ML model coordinate system
   - Need to convert bounding boxes between them

3. **Why DynamicImage?**
   - Flexible format (RGB, RGBA, grayscale)
   - Compatible with `image` crate ecosystem
   - Easy to save/load for debugging

```rust
//! Image Processing Utilities

use anyhow::Result;
use image::{DynamicImage, GenericImageView, ImageBuffer, Rgb};
use ndarray::{Array, Array4};

/// Resize image to target dimensions
///
/// Uses Lanczos3 filter (high quality, slower)
pub fn resize_image(image: &DynamicImage, width: u32, height: u32) -> DynamicImage {
    image.resize_exact(width, height, image::imageops::FilterType::Lanczos3)
}

/// Convert image to RGB (handle RGBA, grayscale, etc.)
pub fn to_rgb(image: &DynamicImage) -> ImageBuffer<Rgb<u8>, Vec<u8>> {
    image.to_rgb8()
}

/// Convert image to tensor for ML models
///
/// Output shape: [1, 3, height, width]
/// Values normalized to [0, 1]
pub fn image_to_tensor(image: &DynamicImage) -> Result<Array4<f32>> {
    let rgb = to_rgb(image);
    let (width, height) = rgb.dimensions();
    
    let mut array = Array4::<f32>::zeros((1, 3, height as usize, width as usize));
    
    for y in 0..height {
        for x in 0..width {
            let pixel = rgb.get_pixel(x, y);
            // Normalize to [0, 1]
            array[[0, 0, y as usize, x as usize]] = pixel[0] as f32 / 255.0;
            array[[0, 1, y as usize, x as usize]] = pixel[1] as f32 / 255.0;
            array[[0, 2, y as usize, x as usize]] = pixel[2] as f32 / 255.0;
        }
    }
    
    Ok(array)
}

/// Normalize tensor for specific model
///
/// Different models expect different normalization:
/// - ImageNet: mean=[0.485, 0.456, 0.406], std=[0.229, 0.224, 0.225]
/// - PaddleOCR: (x / 255.0 - 0.5) / 0.5
pub fn normalize_tensor(
    mut tensor: Array4<f32>,
    mean: [f32; 3],
    std: [f32; 3],
) -> Array4<f32> {
    for c in 0..3 {
        let slice = tensor.slice_mut(s![.., c, .., ..]);
        slice.mapv_inplace(|x| (x - mean[c]) / std[c]);
    }
    tensor
}
```

## Step 3: Add Geometry Utilities

Create `src/utils/geometry.rs` for bounding box operations:

```rust
//! Geometry Utilities
//!
//! Bounding box operations, coordinate transformations, etc.

use serde::{Serialize, Deserialize};

/// Bounding box in pixel coordinates
///
/// Format: (x1, y1, x2, y2) where:
/// - (x1, y1) is top-left corner
/// - (x2, y2) is bottom-right corner
#[derive(Debug, Clone, Copy, Serialize, Deserialize)]
pub struct BBox {
    pub x1: f32,
    pub y1: f32,
    pub x2: f32,
    pub y2: f32,
}

impl BBox {
    /// Create new bounding box
    pub fn new(x1: f32, y1: f32, x2: f32, y2: f32) -> Self {
        Self { x1, y1, x2, y2 }
    }

    /// Width of bounding box
    pub fn width(&self) -> f32 {
        self.x2 - self.x1
    }

    /// Height of bounding box
    pub fn height(&self) -> f32 {
        self.y2 - self.y1
    }

    /// Area of bounding box
    pub fn area(&self) -> f32 {
        self.width() * self.height()
    }

    /// Center point (x, y)
    pub fn center(&self) -> (f32, f32) {
        ((self.x1 + self.x2) / 2.0, (self.y1 + self.y2) / 2.0)
    }

    /// Convert from points to pixels
    ///
    /// # Arguments
    /// * `dpi` - Dots per inch (rendering resolution)
    pub fn from_points(x1: f32, y1: f32, x2: f32, y2: f32, dpi: u16) -> Self {
        let scale = dpi as f32 / 72.0;
        Self {
            x1: x1 * scale,
            y1: y1 * scale,
            x2: x2 * scale,
            y2: y2 * scale,
        }
    }

    /// Convert to points from pixels
    pub fn to_points(&self, dpi: u16) -> Self {
        let scale = 72.0 / dpi as f32;
        Self {
            x1: self.x1 * scale,
            y1: self.y1 * scale,
            x2: self.x2 * scale,
            y2: self.y2 * scale,
        }
    }
}

/// Calculate Intersection over Union (IoU)
///
/// IoU measures overlap between two boxes:
/// - IoU = 1.0: Perfect overlap
/// - IoU = 0.0: No overlap
/// - IoU > 0.5: Significant overlap (common threshold)
pub fn calculate_iou(box1: &BBox, box2: &BBox) -> f32 {
    // Find intersection rectangle
    let x1 = box1.x1.max(box2.x1);
    let y1 = box1.y1.max(box2.y1);
    let x2 = box1.x2.min(box2.x2);
    let y2 = box1.y2.min(box2.y2);

    // Check if boxes intersect
    if x2 < x1 || y2 < y1 {
        return 0.0;
    }

    // Calculate areas
    let intersection = (x2 - x1) * (y2 - y1);
    let area1 = box1.area();
    let area2 = box2.area();
    let union = area1 + area2 - intersection;

    intersection / union
}

/// Non-Maximum Suppression (NMS)
///
/// Removes duplicate detections by keeping only the highest-confidence
/// box among overlapping boxes.
///
/// # Arguments
/// * `boxes` - Detected bounding boxes with confidence scores
/// * `iou_threshold` - IoU threshold for considering boxes as duplicates
///
/// # Returns
/// Filtered list of boxes with duplicates removed
pub fn non_max_suppression(
    mut boxes: Vec<(BBox, f32)>,  // (bbox, confidence)
    iou_threshold: f32,
) -> Vec<(BBox, f32)> {
    // Sort by confidence (descending)
    boxes.sort_by(|a, b| b.1.partial_cmp(&a.1).unwrap());

    let mut keep = Vec::new();
    let mut suppressed = vec![false; boxes.len()];

    for i in 0..boxes.len() {
        if suppressed[i] {
            continue;
        }

        keep.push(boxes[i]);

        // Suppress overlapping boxes with lower confidence
        for j in (i + 1)..boxes.len() {
            if suppressed[j] {
                continue;
            }

            let iou = calculate_iou(&boxes[i].0, &boxes[j].0);
            if iou > iou_threshold {
                suppressed[j] = true;
            }
        }
    }

    keep
}
```

## Step 7: Performance Optimization

### 7.1: Parallel Page Processing

For multi-page documents, render pages in parallel:

Add to `pdf_parser.rs`:

```rust
use rayon::prelude::*;

impl PdfParser {
    /// Parse PDF pages in parallel
    ///
    /// Significantly faster for large documents.
    /// Uses all available CPU cores.
    pub fn parse_to_pages_parallel(
        &self,
        path: impl AsRef<Path>,
        dpi: u16,
    ) -> Result<Vec<PageData>> {
        let document = self.load_document(&path)?;
        let pages = document.pages();
        let page_count = pages.len();
        
        tracing::info!("Processing {} pages in parallel at {} DPI", page_count, dpi);
        
        // Process pages in parallel
        let page_data: Result<Vec<_>> = (0..page_count)
            .into_par_iter()  // Parallel iterator
            .map(|idx| {
                // Each thread gets its own document handle
                let doc = self.load_document(&path)?;
                let page = doc.pages().get(idx as u16)?;
                
                let image = self.render_page(&page, dpi)?;
                let text = self.extract_page_text(&page).ok();
                
                Ok(PageData {
                    page_idx: idx as usize,
                    image,
                    width_pt: page.width().value,
                    height_pt: page.height().value,
                    text,
                })
            })
            .collect();
        
        page_data
    }
}
```

**Benchmark:**
- Sequential: ~500ms per page
- Parallel (8 cores): ~100ms per page effective

### 7.2: Memory-Efficient Streaming

For very large PDFs, process pages one at a time:

```rust
impl PdfParser {
    /// Process PDF pages with callback (memory-efficient)
    ///
    /// Calls callback for each page, doesn't store all pages in memory.
    pub fn process_pages_streaming<F>(
        &self,
        path: impl AsRef<Path>,
        dpi: u16,
        mut callback: F,
    ) -> Result<()>
    where
        F: FnMut(PageData) -> Result<()>,
    {
        let document = self.load_document(&path)?;
        let pages = document.pages();
        
        for (idx, page) in pages.iter().enumerate() {
            let image = self.render_page(&page, dpi)?;
            let text = self.extract_page_text(&page).ok();
            
            let page_data = PageData {
                page_idx: idx,
                image,
                width_pt: page.width().value,
                height_pt: page.height().value,
                text,
            };
            
            callback(page_data)?;
        }
        
        Ok(())
    }
}
```

**Usage:**
```rust
parser.process_pages_streaming("large.pdf", 150, |page| {
    // Process page immediately
    process_ml_models(&page)?;
    // Page is dropped after callback, freeing memory
    Ok(())
})?;
```
